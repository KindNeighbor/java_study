# Chapter 30 Java 7 changed

## 1. JDK 7 이상에서 2진 숫자를 나타내기 위해서 숫자 앞에 추가해 주어야 하는 것은 무엇인가요?

    JDK 7에서 2진 데이터를 표시하려면 0b를 붙여주면 된다.
    
## 2. JDK7 이상에서 숫자의 가독성을 좋게 하기 위해서 숫자 사이에만 적어 놓을 수 있는 특수문자는 무엇인가요?

    JDK 7에서 숫자 값 사이에 Underbar(_)를 사용할 수 있다. 
    이 _ 는 전혀 프로그램상의 의미는 없고, 소스의 가독성을 높이기 위해서 사용된다.
    
## 3. JDK 7이상에서 switch 문에 사용할 수 타입에는 어떤 것들이 있나요? 

    DK 7에서는 String 도 switch-case 문에서 사용할 수 있다.
    
## 4. JDK7에서 추가된 제공하는 다이아몬드는 어떤 상황에서 사용할 수 있나요?

    제네릭(Generic)을 선언할 때 변수 선언부에 제네릭 타입을 사용했다면, 
    객체를 생성하는 부분에는 꺽쇠를 열고 닫는 Diamond를 사용할 수 있다.
    
## 5. JDK7 이상에서 사용할 수 있는 catch 문장은 어떤 것이 달라졌으며, 그 달라진 부분을 적용하기 위해서 사용되는 특수문자는 무엇인가요?

     여러 개의 예외를 하나의 catch 블록에서 사용하려면 Pipe (|)로 여러 예외 타입을 나열할 수 있다.
     
## 6. try 블록 시작 전에 소괄호로 필요한 객체를 생성하는 것을 전문 용어로 무엇이라고 하나요? 

    try-with-resource 를 사용하여 close() 메소드를 사용하여 닫을 필요가 있는 클래스들을 선언할 수 있다.
    이 선언을 해 놓으면 별도로 finally 블록에서 처리해 줄 필요가 없다.
    
## 7. 자바에서 리소스를 닫기 위해서 close() 메소드를 호출해야하는 클래스에 대한 자동 처리를 해준다는 것을 보장하는 인터페이스 이름은 무엇인가요? 


    try-with-resource 를 사용하여 close() 메소드를 사용하여 닫을 필요가 있는 클래스들을 선언할 수 있다.
    이 선언을 해 놓으면 별도로 finally 블록에서 처리해 줄 필요가 없다.
    
    
# Chapter 31 Java 7 added

## 1. JDK 7에서 새로 추가된 Fork/Join은 어떤 것을 의미하나요?

    Fork/Join 이라는 것은 어떤 계산 작업을 할 때 여러 개로 나누어 계산한 후 결과를 모으는 작업을 의미한다.
    
## 2. Fork/Join에 있는 Work steal 이라는 개념은 무엇인가요? 

    Work steal은 작업의 여유가 있는 Queue에서 작업이 많은 Queue의 작업들을 가져가서 일을 하는 것을 의미한다.
    
## 3. Path 라는 인터페이스는 어떤 기능을 제공하나요? 

    Path 인터페이스는 파일과 경로에 대한 정보를 갖고 처리하는데 사용된다.
    
## 4. Files 클래스는 어떤 기능을 제공하나요?

    Files 클래스는 Path 객체를 사용하여 파일을 통제하는데 사용된다.
    
## 5. WatchService 라는 클래스는 어떤 기능을 제공하나요?

    WatchService 클래스는 파일의 변경을 다루기 위해서 사용된다.
    
## 6. NIO에서 이야기하는 채널(Channel)은 무슨 역할을 하나요?

    NIO의 Channel 은 데이터를 처리하기 위한 통로를 제공한다.
    
## 7. java.util 패키지에 추가된 Objects 라는 클래스는 어떤 메소드들을 제공하나요?

    Objects라는 클래스는 Object 클래스가 제공하는 기능들을 static으로 제공하지만, 매개변수로 넘어오는 객체가 null 이라고 할지라도 예외를 발생시키지 않도록 구현해 놓았다.
    
# Chapter 34 JavaFX

## 1. JavaFX를 자바 기반의 어떤 라이브러리라고 하나요 ?

    JavaFX 는 자바 기반의 GUI 라이브러리이다.
    
## 2. 레이아웃을 구성하는 XML 파일을 무엇이라고 부르나요? 

    FXML 이라는 레이아웃을 위한 XML 파일을 사용하며 Scene Builder 로 쉽게 화면을 구성할 수도 있다.
    
## 3. JavaFX에서 웹 페이지를 브라우징 할 수 있도록 어떤 엔진을 내장하고 있나요?

    JavaFX는 자체적으로 웹 페이지를 브라우징할 수 있는 웹 엔진을 내장하고 있다.
    
## 4. JavaFX를 구성하는 기본 구성요소 3가지는 무엇인가요?

    avaFX는 기본적으로 Main 클래스, Controller 클래스, FXML 파일 세개만 있으면 기본적인 애플리케이션을 구동할 수 있다.
    
## 5. 화면의 배치를 위한 클래스들을 무엇이라고 하나요 ?

    화면의 레이아웃을 처리하기 위한 레이아웃 클래스들이 존재한다.
    
## 6. 화면을 구성하는 구성요소들을 무엇이라고 하나요 ?

    화면을 구성하는 UI 컴포넌트들이 존재한다.
    
## 7. 사용자가 버튼 같은 것을 눌렀을 때 어떤 처리를 해 줘야 하나요 ?

    UI 컴포넌트의 액션을 처리하는 이벤트를 구현해야 UI 컴포넌트가 동작한다.
